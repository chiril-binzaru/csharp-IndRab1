Разработайте программное приложение, которое позволяет управлять
мероприятиями (например: конференции, воркшопы, соревнования, meet-up встречи) и
участниками, используя базы данных, специализированные элементы управления и
отчёты.
Приложение должно обеспечивать:
1. ввод и управление данными
2. отображение и фильтрацию информации
3. использование хранимых процедур, представлений и транзакций
4. генерацию релевантных отчётов
Цели
Выполняя данную работу, вы:
1. будете использовать элементы управления для работы с базой данных
2. научитесь создавать и управлять объектами базы данных
3. реализуете хранимые процедуры, представления и транзакции
4. будете генерировать параметризованные отчёты
5. разработаете полноценное приложение с функциональным интерфейсом
ОБЩИЕ ТРЕБОВАНИЯ
Спроектируйте и разработайте настольное приложение (Windows Forms),
интегрированное с системой управления реляционными базами данных (например:
MySQL или SQL Server), которое обеспечит полное администрирование данных через
интерактивный графический интерфейс.
Приложение должно реализовывать эффективные механизмы:

1. Проектирование базы данных
	Спроектируйте и реализуйте реляционную базу данных, включающую минимум 3–
	4 взаимосвязанные таблицы, необходимые для работы приложения.
	Структура базы данных должна включать следующие сущности:
	1. Таблица «Мероприятия»
		1. ID_мероприятия (первичный ключ)
		2. название
		3. дата
		4. место проведения
		5. тип_мероприятия
	2. Таблица «Участники»
		1. ID_участника (первичный ключ)
		2. имя
		3. фамилия
		4. email
	3. Таблица «Регистрации»
		1. ID_регистрации (первичный ключ)
		2. ID_мероприятия (внешний ключ)
		3. ID_участника (внешний ключ)
		4. статус (например: подтверждён / не подтверждён)
	4. Дополнительная таблица «Организаторы» (опционально)
		1. определите поля самостоятельно
	Дополнительное требование:
	Определите корректные связи между таблицами, используя первичные и внешние
	ключи, обеспечивая ссылочную целостность данных.
	Рекомендуемые связи:
	1. 1:N между Мероприятиями и Регистрациями
	2. 1:N между Участниками и Регистрациями

2. Объекты базы данных
	Создайте и используйте:
	1. минимум 2 хранимые процедуры (например: добавление участника,
	регистрация на мероприятие)
	2. минимум 1 представление (VIEW) (например: список участников по
	мероприятиям)
	3. минимум 1 транзакцию (например: регистрация + обновление количества мест)

3. Интерфейс приложения (элементы управления)
	Разработайте графический интерфейс (Windows Forms), обеспечивающий полное
	взаимодействие с данными, используя элементы управления и механизм привязки данных
	(Data Binding).
	1. Обязательные элементы интерфейса
		1. Отображение данных:
			a. DataGridView — для отображения таблиц:
			b. Мероприятия
			c. Участники
			d. Регистрации (включая объединённые данные, например: имя участника +
			название мероприятия)
		2. Ввод и редактирование данных:
			a. TextBox для:
				i. названия мероприятия
				ii. места проведения
				iii. типа мероприятия
				iv. имени и фамилии участника
				v. email
				vi. DateTimePicker — для выбора даты
			b. ComboBox для:
				i. выбора мероприятия
				ii. выбора участника
				iii. выбора статуса регистрации
		3. Действия:
			a. Кнопки для:
				i. Добавления
				ii. Редактирования
				iii. Удаления
				iv. Сброса полей
				v. Поиска / фильтрации
	2. Обязательные функции
		1. Управление мероприятиями:
			1. добавление
			2. редактирование
			3. удаление
			4. отображение
		2. Управление участниками:
			1. добавление
			2. редактирование
			3. удаление
			4. отображение
		3. Управление регистрациями:
			1. создание регистрации (через ComboBox)
			2. установка статуса
			3. отображение (JOIN)
		4. Поиск и фильтрация:
			1. поиск по имени/email
			2. фильтрация по типу/месту
			3. фильтрация по статусу
		5. Привязка данных:
			1. подключение к БД
			2. автообновление таблиц
			3. синхронизация данных
		6. Валидация данных:
		1. проверка обязательных полей
		2. проверка email
		3. предотвращение ошибок
	Дополнительно:
	Организуйте интерфейс логично (вкладки или секции)

4. Работа с данными
	Реализуйте:
	1. CRUD операции
	2. использование процедур
	3. работу с представлениями
	4. валидацию

5. Отчёты
	Разработайте модуль отчётов.
	Требования:
	1. Минимум 2 отчёта:
	a. список участников по мероприятию
	b. количество участников
	c. мероприятия за период
	2. Функциональность:
	a. минимум 1 параметризованный отчёт
	b. использование JOIN
	c. понятный формат
	3. Технологии:
	a. ReportViewer (RDLC)
	b. или аналог
	4. Интеграция:
	a. выбор параметров
	b. отображение
	c. обновление данных
